## 1. Описание системы

Проект представляет собой охранную сигнализацию на микроконтроллере ESP32 с удаленным управлением через Telegram Bot. Система подключается к Wi-Fi, синхронизирует время по NTP и отслеживает движение в помещении. Присутствует возможность локального управления физической кнопкой.

### Ключевые возможности:

* Асинхронное управление через Telegram Bot.
* Синхронизация времени (NTP) и работа по расписанию.
* Программный антидребезг контактов кнопки.
* Неблокирующая работа цикла (использование `millis()`).
* Автоматические уведомления о срабатывании и смене статусов.

## 2. Режимы работы (Конечный автомат)

Система работает на базе четырех состояний:

| Состояние | Описание |
| --- | --- |
| **IDLE** | Ожидание. Опрос датчика движения и периферия отключены. |
| **ARMED** | Охрана активна. Ведется опрос PIR-датчика. При фиксации движения (LOW → HIGH) система переходит в режим тревоги. |
| **ALARM** | Тревога. Светодиод и зуммер работают с частотой переключения 250 мс. Отправлено уведомление в Telegram. После истечения таймаута — переход в режим отката. |
| **COOLDOWN** | Пауза (10 секунд) перед возобновлением охраны для исключения ложных и повторных срабатываний. |

## 3. Аппаратная часть и схема подключения

**Компоненты:** ESP32 DevKit C v4, PIR-датчик движения (HC-SR501), датчик DHT22, пьезобуззер, тактовая кнопка, красный светодиод.

| Пин ESP32 | Подключение | Описание / Назначение |
| --- | --- | --- |
| **GPIO 13** | PIR_PIN | Вход сигнала движения (`INPUT_PULLDOWN`) |
| **GPIO 2** | LED_PIN | Выход на светодиод через резистор 220 Ом |
| **GPIO 15** | BUZZER_PIN | Выход на зуммер (управление через `tone()`) |
| **GPIO 4** | BUTTON_PIN | Физическая кнопка сброса (`INPUT_PULLUP`) |
| **GPIO 5** | DHT_PIN | Шина данных датчика температуры/влажности |
| **3V3** | VCC | Питание PIR и DHT22 |
| **GND** | GND | Общая земля всех компонентов |

## 4. Программная часть

Список зависимостей (`libraries.txt`):

* `WiFi.h`, `WiFiClientSecure.h` — сетевое подключение и TLS-шифрование.
* `UniversalTelegramBot` — взаимодействие с Telegram API (polling).
* `ArduinoJson` — разбор JSON-ответов.
* `DHT sensor library for ESPx` / `DHT sensor library` — получение данных о климате.
* `time.h` (stdlib) — получение и форматирование времени по NTP.

## 5. Команды Telegram Bot

| Команда | Действие |
| --- | --- |
| `/arm` | Включить охрану |
| `/disarm` | Выключить охрану (сброс состояния) |
| `/status` | Вывод текущего режима, времени, температуры, влажности и настроек |
| `/alarm on` / `off` | Включить или отключить звуковую сирену |
| `/alarm time {сек}` | Установить длительность тревоги (сек) |
| `/noise {0-100}` | Задать громкость сирены (в процентах) |
| `/schedule HH:MM HH:MM` | Установить расписание авто-включения и выключения |
| `/schedule off` | Деактивировать расписание |
| `/help`, `/start` | Вывести список команд |
