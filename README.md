ESP32 Telegram Smart Security System — это легковесная, автономная система IoT-сигнализации (Headless IoT), управляемая исключительно через мессенджер Telegram и с дополнительной возможностью настроить систему через Веб-Интерфейс. Проект оптимизирован для работы в условиях ограниченных ресурсов микроконтроллера и лимитированного обьёма процессов облачных серверов.
Система отслеживает движение с помощью PIR-датчика, включает световую и звуковую сигнализацию, мониторит климат (DHT22) и поддерживает настройку работы по расписанию.
Основной функционал
Управление через Telegram: Полный контроль и настройка системы через бота. Он так же служит и логом системы
Настройка через web-интерфейс: Дополнительная возможность настроить систему через webserver.
Изолированная безопасность: Бот реагирует только на сообщения от заданного пользователя.
Машина состояний (FSM): Надежная логика переходов: IDLE ➔ ARMED ➔ ALARM ➔ COOLDOWN.
Прочее: 
Интеграция с датчиком DHT22 для вывода текущей температуры и влажности. 
Настройки автоматической постановки и снятия с охраны (сверка времени по NTP).
Дополнительная кнопка  для экстренного отключения сирены.

Архитектура: Машина состояний (State Machine)
Система имеет четыре переключаемых рабочих состояния:
IDLE (Ожидание): Охрана отключена. Система реагирует только на Telegram-команды и сверяет расписание.
ARMED (Под охраной): Система активно опрашивает PIR-датчик. Фиксируется только фронт сигнала (LOW ➔ HIGH).
ALARM (Тревога): Активация сирены и стробоскопа. Отправка HTTP POST запроса в API Telegram. Переход в следующее состояние по истечении alarmDuration или по физической кнопке.
COOLDOWN: Пауза, необходимая для калибровки PIR-датчика и защиты от спама уведомлениями в Telegram. После завершения система возвращается в статус ARMED. (Длительность 10 секунд).
